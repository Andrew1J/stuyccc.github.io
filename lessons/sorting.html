Sorts¶

Sorting algorithms are algorithms used to put arrays into order. There are tons of sorting algorithms, all with different use cases and time complexities. Some may have the same time complexity, but perform wildly differently depending on the input data's properties and other variables. We'll be explaining a few of the most common and useful sorting algorithms here.

Merge Sort¶

Merge sort works by splitting the input array into "chunks" recursively until each "chunk" is 2 integers long, this process is often called a "divide and conquer" approach. It then sorts each of these chunks, by checking which number is greater. Finally, it recursively merges the chunks, preserving the order, until the input array has been reassembled.

Merge Sort runs in O(nlogn) on average

Quick Sort¶

Quick Sort is another divide and conquer sorting algorithm. Quick Sort uses a pivot to split the array, placing all elements that are greater than the pivot to the right, and smaller to the left. Then, each sub-array has Quick Sort run recursively on it.

While Quick Sort has the same upper bound as Merge Sort, the lower bound of the time complexity can be lower. We still say that it runs in O(nlogn).

Binary Search¶

When you need to find an element in an array, you can either iterate over the array until you find the element, or you can employ the use of a search algorithm. One of these search algorithms is binary search. Binary search operates on sorted arrays. It works by checking the value at the middle of the array. If the middle is greater than the value you're searching for, you re-run binary search on the first half of the array. If it's less than the value you're searching for, do the same, but run it on the second half of the array.

The runtime of a binary search is O(logn).
