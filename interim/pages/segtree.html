SEGMENT TREES

Motivation

Let's imagine the following problem, where we start with a given array A and want to process the following queries:

1: set the value at index K to X.

2: given indices L and R, calculate A[L] + A[L+1] + ... + A[R]

If the array has length N and there are Q queries, we can easily solve this problem in O(NQ) by doing what the problem tells us to do. But we can do better!

The simple approach is inefficient because it has to process every element in the range. So let's try recording the sum of adjacent pairs of numbers as well. Using this method, we can calculate the sum potentially twice as fast. This is still O(NQ), however, so we can do better.

The key is that we can keep doing this extra calculation for every power of two. We can calculate segments of 4 numbers by summing the segments of 2 numbers below it, do the same for 8, and so on. The following diagram describes this:
