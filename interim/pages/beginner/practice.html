Problem 1

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int w; cin >> w;
    if (w % 2 == 1 || w == 2) {
        cout << "NO\n";
    } else {
        cout << "YES\n";
    }
}

Explanation

We need to split the melon into 2 evens parts, first observation is that no odd number can be split into 2 even numbers, so all odd numbers return NO. 2nd observation is that any even number works except 2, because all even numbers except 2 can be split into n/2 and n/2 which are both even. So our final answer is if n is odd or 2 return NO else return YES.

Problem 2

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t; cin >> t;
    int ans = 0;
    while (t--) {
        int a, b, c; cin >> a >> b >> c;
        if (a+b+c >= 2) ans++;
    }
    cout << ans << "\n";
}

Explanation

We are given t contests and we need to figure out the number of contests where they write a solution. Notice that they write a solution when at least 2 of them are confident in their answers. We are given 3 numbers (a,b,c) where they equal 1 if they are confident and they equal 0 if they are not confident. Notice that they solve the problem if (a+b+c) >= 2, because this means at least 2 values are 1, which is what we want. Now if this condition is true, we add 1 to a counter, and after the t contests, we print the counter.

Problem 3

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int k, l, m, n; cin >> k >> l >> m >> n;
    int d; cin >> d;
    int ans = 0;
    for (int i = 1; i <= d; i++) {
        if (i % k == 0 || i % l == 0 || i % m == 0|| i % n == 0) ans++;
    }
    cout << ans << "\n";
}

Explanation

Notice that if we number our dragons from 1 to d, then the k attack would hit the k dragon, then the 2*k dragon, then the 3*k dragon, and so on. This means we know that if a dragon is divisible by an attack k, then it'll get hit by it. And our goal is to find the number of dragons that have been hit by at least one attack. So we can just loop from 1 to d, and check if each dragon is divisible by any of the 4 numbers given and if that is the case we increase a counter by 1, else we do nothing. And after the loop is finished, we output the counter.

Problem 4

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t; cin >> t;
    int mx = 0, curr = 0;
    while (t--) {
        int a, b; cin >> a >> b;
        curr -= a;
        curr += b;
        if (curr > mx) mx = curr;
    }
    cout << mx << "\n";
}

Explanation

Notice that at every train stop, after people get off and get on, there is a certain number that remain on the train, and the trains capacity must be able to fit all these people that remain on the train. This means that the mx(capacity we need) >= curr(current capacity) for every train stop, and if we want to find the minimum maximum capacity this means we set mx to the largest curr value we find and no more than that. Now we can see that at the beginning we can set curr = 0 (no one is on the train), and mx = 0 (maximum of all currs so far is 0 because curr is 0), and then as we go through every stop we update curr (curr -= a, curr += b), and then if this curr becomes bigger than mx we have to update mx to reflect that, but if it doesn't become bigger than mx we don't need to change anything, and we can continue on with the loop.

Problem 5

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int k, r; cin >> k >> r;
    for (int i = 1; i <= 10; i++) {
        if ((k*i) % 10 == r || (k*i) % 10 == 0) {
            cout << i << "\n";
            break;
        }
    }
}

Explanation

First observation for this problem is that Polycarp does not want to buy shovels if there is change given back to him, and remember tha the only has 10 coins and a single r coin. So if he doesn't want any change that means the price of the shovels he wants to buy (lets call it x) has to have at least one of the following properties: x % 10 == 0 or x % 10 == r. Notice in the first case he can just use all 10 coins and not get any change, while in the second case he can use a singular r coin and then use 10 coins for the rest of the purchase.

Now we just have to figure out the smallest x, that satisfies one of these conditions. Notice that x = i*k, where k is the price of 1 shovel and i is the number of shovels being bought. So now we need to find the smallest i, and to do this we can loop from i = 1 to i = 10, and print the first i that satsifies these conditions. But why is our maximum 10? Our maximum is 10 because it is gauranteed htat 10*k % 10 == 0 for any k, so this means that 10 always works so we have no need to go past 10 to check larger values of i that work.

Problem 6

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t; cin >> t;
    int c1 = 0, c2 = 0;
    while (t--) {
        int a, b; cin >> a >> b;
        if (a > b) c1++;
        if (a < b) c2++;
    }
    if (c1 > c2) cout << "Mishka\n";
    else if (c1 == c2) cout << "Friendship is magic!^^\n";
    else cout << "Chris";
}

Explanation

We have to loop through t games, and see who wins each game and count total wins. To do this we can keep an counter c1 to count Mishka's wins, and a counter c2 to count Chris's wins, and update these counters to reflect the winners of each game as described in the problem. Once we have these final counts, if c1 > c2, we print "Mishka", if c1 == c2, we print "Friendship is magic!^^", and else we print "Chris".

Problem 8

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t; cin >> t;
    while (t--) {
        int n, m; cin >> n >> m;
        if (2*n > m) {
            cout << -1 << " " << -1 << "\n";
            continue;
        }
        cout << n << " " << 2*n << "\n";
    }
}

Explanation

The optimal strategy for this problem is to make x = l, because the smaller x is the more multiples of x(ie x*2,x*3,x*4) that can be found the in range [l,r]. Now that we have x = l, the smallest multiple of x that is in the range and isn't x itself (x can't be lcm(x,y), because y > x from the problem) is 2*x. Since this is the smallest value of multiple possible, if this value is in the range we can just return x and 2x because the lcm of (x,2x) = 2x, However, if 2x is not in the range, then no other multiple of x is possible, and so just return -1 -1.

Problem 9

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t; cin >> t;
    while (t--) {
        int n; cin >> n;
        if (n <= 3) {
            cout << n-1 << "\n";
        } else if (n % 2 == 0) {
            cout << 2 << "\n";
        } else {
            cout << 3 << "\n";
        }
    }
}

Explanation

Lets try to split this up into cases, first lets find a pattern, if n is even we can divide by n/2, get 2, and then subtract 1 to get 1, so for evens it looks like the answer is always 2. However, this does not work for all evens, we have the case of n=2, where the solution is 1 instead of 2. For odds, we can see that we can subtract 1 to get an even then do the same process as for evens and get the answer seems like it is always 3. Except for n=1 and n=3, this always works because they only need 0 and 2 moves for this problem. Anyhow now that we have the cases we can say if n <= 3 the answer is n-1, else if its even the answer is 2, else if its odd the answer is 3.

Problem 10

Solution 

#include <bits/stdc++.h>
using namespace std;

int main() {
    // note if we have two integers x and y.
    // ceil(x/y) = (x+y-1)/y.
    // Try to figure out why this is true yourself.
    int k, n, s, p;
    cin >> k >> n >> s >> p;
    int x = k * ((n+s-1) / s);
    cout << ((x+p-1) / p) << "\n";
}

Explanation

Each person has to make n planes and they can make s per sheet, also people can't share paper so this means each person need ceil(n/s) sheets of paper for their airplanes. There are k people, so we need k*ceil(n/s) sheets, lets call this x for short. Each packet of paper has p sheets of paper, so our final answer is ceil(x/p) packets of paper are needed.
